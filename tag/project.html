H.264的零散笔记整理

Date 2014-12-15 By shilei Category 视频传输. Tags project

H264与ffmpeg

简单地说，H.264是一种编码协议，而ffmpeg则是含有支持H.264编码程序的程序包。H.264压缩比非常高，所以是目前音视频压缩的主流算法。现在已经有了H.265压缩算法，理论上在相同的带宽下可以传输两倍的数据量，并且已经有了不少成熟的产品开始投入使用。

H.264中帧，场，宏块等数据单位

一帧，即一副静止的图像，可分为两场（奇数场和偶数场）。

或者说一个视频图像编码后的数据叫做一帧（frame）
一帧有一个片或者多个片（slice）组成
一个片由一个或多个宏块（MB）组成
一个宏块由16×16的YUV数据组成

视频编码算法以宏块为单位，逐个编码，组成连续视频流

帧又可以分为：I帧，P帧，B帧
I帧： 也叫做关键帧，包含完整的画面信息
P帧：前向预测帧，只存储与前一个已解压帧的差值
B帧：双向预测帧，同时依赖前一个帧和后一个帧的编解码

一. 客户端相关cpp的简要说明
  • main.cpp ：其中包括字符格式的设置，QSplashSreen实现Qt程序的画面启动等。
  • common.h : 定义程序的通用信息，即一些宏定义。
  • abstractconf.cpp : 设置类的基类。比如虚函数 loadConf(),saveConf()...
  • Uiappabout : 显示程序关于信息，包括软件的作者学校等信息。
  • Uiappoutput.cpp : 整个程序运行中相关信息的显示输出类
  • Uidatasingle.cpp : 视频连接界面。有显示网络节点，新建及放弃连接等成员函数
  • Uivideowindow.cpp : 视频渲染底板。本程序提供了6个视频显示窗口实现addWin(),delWin()添加删除渲染空间。
  • Uimainconf.cpp : 设置界面集合。addConf()增加相关配置界面，此程序中目前有程序，路由，视频和扩展4个配置界面。confUpdate()完成更新。applyClicked()应用配置修改。还有一些其他相关配置参数的操作。
