RTP相关笔记

Date 2014-12-17 By shilei Category 视频传输.

为什么采用RTP而不直接利用UDP发送数据

UDP无法加载流的时间信息，而RTP可以提供时间信息并且实现流同步。

H.264 RTP 负载格式

NALU

NALU头由一个字节组成
+-------------------------------+
| 0 | 1 | 2 | 3 | 4 | 5 | 6 | 7 |
| F | NRI | 　　TYpe 　|　　

F : 在Ｈ.264中规定必须置为1
NRI : 知道NALU重要性，一般情况不关心
Type：指示这个NALU单元的类型。
NALU使用RTP包进行发送的可能类型有三种，都由Type字段相应的值进行指定。
1. 单一NAL单元模式，即一个RTP包仅由一个完整的NALU组成。 2. 组合封包模式，即可能由多个NAL单元组成一个RTP包。 3. 分片封包模式，用与把 一个NALU单元封装成多个RTP包。

在我的使用中，只遇到了第一种和第三种类型

NALU长度小于MTU大小的包，一般采用这种发送方式。

H.264的零散笔记整理

Date 2014-12-15 By shilei Category 视频传输. Tags project

H264与ffmpeg

简单地说，H.264是一种编码协议，而ffmpeg则是含有支持H.264编码程序的程序包。H.264压缩比非常高，所以是目前音视频压缩的主流算法。现在已经有了H.265压缩算法，理论上在相同的带宽下可以传输两倍的数据量，并且已经有了不少成熟的产品开始投入使用。

H.264中帧，场，宏块等数据单位

一帧，即一副静止的图像，可分为两场（奇数场和偶数场）。

或者说一个视频图像编码后的数据叫做一帧（frame）
一帧有一个片或者多个片（slice）组成
一个片由一个或多个宏块（MB）组成
一个宏块由16×16的YUV数据组成

视频编码算法以宏块为单位，逐个编码，组成连续视频流

帧又可以分为：I帧，P帧，B帧
I帧： 也叫做关键帧，包含完整的画面信息
P帧：前向预测帧，只存储与前一个已解压帧的差值
B帧：双向预测帧，同时依赖前一个帧和后一个帧的编解码

使用Pelican和Github Pages搭建个人博客

Date 2014-11-03 By shilei Category 教程. Tags github
前言

近段时间有了想搭建个人博客的冲动，因此也看了不少利用Pelican和Github Pages搭建博客的教程，但是完全按照教程去做，总会出现一些教程中没有提到的问题。因此，把自己成功搭建的过程也分享一下。

基础

1.搭建博客之前最好掌握git的基本操作，这样会让你在搭建的时候少很多困惑，比如一些基本的git命令，知道它的作用会对搭建博客以及以后博客的更新起到很大的作用。在此强烈推荐progit，不需要看完整本书，只需要了解其基本的操作命令。电子版链接progit

2.pelican和github可自行搜索相关资料进行了解。博客写作编辑器markdown自行学习1小时便能搞定。

ubuntu 14.04
首先linux内核的版本就是利用git进行版本控制的，在ubuntu环境搭建基于github的博客系统个人感觉更加简单，更容易上手，用起来也更加方便，当然这得有linux环境操作基础。

安装Pelican
pip install pelican

系统一般没有预装pip，因此需要先把pip安装了
