AdHocMonitor系统中关于视频部分的说明

Date 2014-11-06 By shilei Category technology.

AdHocMonitor系统中关于视频部分的说明

一. 客户端相关cpp的简要说明
  • main.cpp ：其中包括字符格式的设置，QSplashSreen实现Qt程序的画面启动等。
  • common.h : 定义程序的通用信息，即一些宏定义。
  • abstractconf.cpp : 设置类的基类。比如虚函数 loadConf(),saveConf()...
  • Uiappabout : 显示程序关于信息，包括软件的作者学校等信息。
  • Uiappoutput.cpp : 整个程序运行中相关信息的显示输出类
  • Uidatasingle.cpp : 视频连接界面。有显示网络节点，新建及放弃连接等成员函数
  • Uivideowindow.cpp : 视频渲染底板。本程序提供了6个视频显示窗口实现addWin(),delWin()添加删除渲染空间。
  • Uimainconf.cpp : 设置界面集合。addConf()增加相关配置界面，此程序中目前有程序，路由，视频和扩展4个配置界面。confUpdate()完成更新。applyClicked()应用配置修改。还有一些其他相关配置参数的操作。
  • Uivideocons.cpp : 视频连接列表界面组件。实现删除，清空列表。比如双击连接的IP地址，即可删除对应此IP的视频连接。
  • Uivideoconf.cpp : 视频配置界面组件。其中包括一些端口的设置，刷新时间、确认时间、最大连接数、是否自动重连等参数的设置。
  • Uivideorender.cpp : 视频画面渲染类。客户端将接收到的画面数据进行渲染，最终显示在屏幕上。
  • Uivideosingle.cpp : 独立新建视频连接。添加选择节点addNode()，双击处理单元 nodeSelect()。以及新建独立视频连接的相关实现。
  • Uivideowin.cpp : 单视频画面窗口。其中包括了解码、渲染等相关类。
  • decodethread.cpp : 解码线程。
  • videocom.cpp : 视频通讯控制类。获取视频配置界面的参数，并且通过QUdpSocket类和远程端口建立连接，把当前客户端参数设置发送给服务器。
  • videoconf.cpp : 继承了abstractconf类，并且实现了loadConf(),saveConf()等基类中的虚函数。
  • videoins.cpp : 单视频连接。视频配置界面组件的参数获取（由用户自定义视频传输的参数）。
  • videomain.cpp : 视频控制系统核心。其中包括了，各种组件的的获得，状态的变化insStarted（） 等signal以及视频的显示新建，启动、停止等slot。
二. 服务器端相关cpp的简要说明
  • Common.h : 变量及结构定义。其中包括控制参数结构体，客户端连接队列，客户端信息结构体，缓冲区以及各种视频参数变量的声明定义。
  • H264.cpp : 摄像头的初始化InitCarm(),RTP连接的初始化InitRTPConPara(),以及数据压缩 API的调用。包括TS传输时，pmt，pat，pts，pcr等信息的写入。最后实现两种RTP/TS发送方式的sendMainTS()和sendMainRTP()。
  • H264server.cpp : 负责服务器端的监听。其中有初始化客户端连接队列，初始化连接参数，从配置文件读写取配置参数，接受客户端连接的请求，从客户端队列中移除某个客户端等具体操作。
三.server端的主要运行流程
  1. Init_Para() //初始化传输参数 在h264.conf中调用ReadConfigfile()；
  2. Init_ctrl_property() //初始化控制信息 即_control结构体，内有sysexit sendThread等成员；
  3. Init_client_queue() //初始化客户端连接队列 即结构体_clientQ,内有RTPsession RTPsessionParams等成员；
  4. comu_server_listen() //创建socket连接，申请serverMsgBuffer[]缓冲区接收客户端发送过来的消息 消息中包括消息类型如SOC_CON2,SOC_RES，resolution,framerate,streamtype...然后把消息参数写如h264.conf配置文件中。
接着需要判断刚接收到streamtype类型，如果streamtype=1，表示客户端希望TS传输方式，如果streamtype=2，则表示客户端希望RTP传输方式
TS传输方式,主要步骤在sendMainTS()函数中
a.dump_video_para（）//打印视频传输相关参数`  即从客户端接收的参数消息
b.Init_Carm() //初始化摄像头 其中包括了帧缓冲区的申请，内存映射等操作，最后采集图像。  
c.GetCamrFramNALU() //得到一个NALU，这里实现时默认NALU即一个视频帧，其中还包括了MFC（Multi Fomat Codec,支持H.264的编解码）编码的三个函数，对一帧进行了压缩编码  
d.retransmit_si_info(),mpegts_insert_pcr_only() //插入PMT，PAT表和PCR  
e. prepare_packet_header() //write PES header ,write pts dts  
f. sendto() //往客户端发送数据帧  TS_PACKET_SIZE ： 188字节

######RTP传输方式，主要步骤在`sendMainRTP()`函数中
a.dump_video_para() //打印视频传输相关参数  
b.Init_Carm()  
c.InitRTPConPara() //初始化RTP连接 如增加请求连接的客户端地址  
d.GetCamrFramNALU()和TS一样，通过ioctl(fd,VIDIOC_DQBUF,&buf)从缓冲区中取得        一帧并进行压缩编码。  
 e.根据帧的大小payloadsize判断是否需要分包。然后调用RTP的sendPacket()发送数据帧
四. client流程,主要说明DecodeThread.cpp
判断videostreamtype的值
videostreamtype==2
  1. 创建RTP会话，设置RTP参数
  2. 注册所有容器格式和COEDC，包括avCodeCtx解码器上下文结构，其中包括视频 width,height等
  3. avcodec_open() //打开解码器
  4. session.BeginDataAccess() //开始接收数据
  5. 查看nalutype的值，判断是否分片传输的。如果是分片传输，则需要判断起始片和结束片。
  6. 接收完整NALU后调用avcodec_decode_video()解码
  7. sws_getContext()进行图像转换，把YUV420P(亮度和色彩，节省带宽)转换成RGB32
videostreamtype==1
  1. 注册容器格式和CODEC avcodec_init()
  2. avFormatCtx->streams[count]->codec->codec_type==CODEC_TYPE_VIDEO寻找视频流
  3. avcodec_open() 打开解码器
  4. av_read_frame() 从视频流中读取视频数据
  5. avcodec_decode_video2() 对视频数据进行解码
  6. 进行图像转换
